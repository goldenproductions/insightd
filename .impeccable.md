## Design Context

### Users
Homelabbers managing personal infrastructure (Proxmox VMs, Docker containers, self-hosted services). Technical enough to run servers, but not professional sysadmins. They want to understand what's happening across their systems without needing to become monitoring experts. The tool should teach and surface insights, not just display data. Commercial expansion is a future goal — the UI must feel professional enough to scale beyond hobbyists.

### Brand Personality
**Reliable, minimal, insightful.**

- **Voice**: Calm, confident, helpful. Speaks like a knowledgeable friend, not a control room.
- **Tone**: Reassuring when things are fine, clear and direct when they're not. Never alarmist.
- **Emotional goal**: The user should feel *in control* — a sense of "I know what's going on" without needing to dig. The UI does the thinking, the user gets the understanding.

### Aesthetic Direction
- **Visual tone**: Clean, polished, minimal. Every element earns its place.
- **Theme**: Dark and light mode, slate-based palette. Semantic colors (emerald/amber/red/blue) for status only — not decoration.
- **Anti-reference**: Grafana. No dense graph walls, no configuration overload, no expert-only interfaces. Insightd is the opposite — curated, opinionated, and immediately legible.
- **Polish direction**: Push current design toward more refined spacing, subtle transitions, and considered micro-details. Professional enough to be commercial, approachable enough for a first-time homelabber.

### Design Principles

1. **Clarity over density**: Show the right information at the right time. One clear insight beats ten raw metrics. Resist the urge to add more graphs — instead, add more meaning.

2. **Calm by default, urgent when needed**: The UI should feel quiet and confident when systems are healthy. Reserve visual intensity (color, motion, prominence) for moments that genuinely need attention.

3. **Teach, don't just display**: Help users understand *why* something matters, not just *what* the number is. Context labels, trend indicators, and plain-language insights over raw data dumps.

4. **Progressive disclosure**: Summaries first, details on demand. Every screen should have a clear hierarchy: hero metric or status, supporting context, then expandable details for those who want to dig deeper.

5. **Polish in the details**: Consistent spacing, smooth transitions, considered typography. The difference between "good enough" and "professional" lives in the micro-details — alignment, hover states, loading states, empty states.

### Signature Patterns

- **Metric personalities**: Every metric gets a human-friendly emoji + analogy label relative to its baseline (e.g. CPU: "Napping" → "Burnout", Memory: "Sharp as kid" → "Total blackout"). Baseline-aware tiers compare against the entity's own history (P50 = Normal, P75 = Elevated, P90 = High, P95+ = Critical). This is a core differentiator — it makes numbers *relatable*.
- **Health scores**: 0–100 circular SVG badge with color gradient (green 90+, amber 70–89, orange 50–69, red <50). Used as the hero metric on the dashboard.
- **Unified attention feed**: Alerts, downtime, and insights merged into a single "Needs Attention" list, sorted by severity. No separate alert pages to check.
- **Uptime timelines**: 7-day horizontal bar with hourly slots (green/red/muted). Clickable for incident details.
- **Rating pills**: Percentile-based badges with 20% opacity semantic backgrounds. Consistent across all metric views.

### Design Tokens

**Palette** — Slate-based with semantic accents:
- Success: `#10b981` (emerald-500)
- Warning: `#f59e0b` (amber-500)
- Danger: `#ef4444` (red-500)
- Info: `#3b82f6` (blue-500)
- Orange accent: `#f97316` (orange-500, used for "high but not critical")
- Backgrounds/surfaces/text: CSS variables that swap between light (slate-50 → slate-900) and dark (slate-900 → slate-100) themes

**Typography**:
- Font stack: `-apple-system, BlinkMacSystemFont, 'Segoe UI', 'Inter', Roboto, sans-serif`
- Body: `text-sm` (14px)
- Secondary/meta: `text-xs` (12px)
- Card titles: `text-sm font-semibold uppercase tracking-wide text-secondary`
- Page headings: `text-lg` / `text-xl font-bold`
- Weights: regular (body), medium (labels), semibold (card titles, badges), bold (values, headings)

**Spacing & Radius**:
- Cards: `rounded-xl` (12px), `p-4 lg:p-5`
- Buttons/inputs: `rounded-lg` (8px), `px-4 py-2`
- Badges/pills: `rounded-full`, `px-2 py-0.5`
- Small indicators: `rounded-sm` (2px)
- Gaps follow Tailwind scale: `gap-1` through `gap-5`, prefer `gap-3` for card grids

**Depth & Motion**:
- No box shadows — depth conveyed through borders and background contrast
- Global 150ms transitions on background-color, border-color, color
- Hover: background shift (`.hover-surface`) or opacity (`hover:opacity-80`)
- Status dots: `h-2` (small), `h-2.5` (medium), `h-3` (large)

### Component Conventions

- **Card**: `rounded-xl border border-border bg-surface p-4 lg:p-5` — the universal container. Optional uppercase title.
- **StatCard**: Card with large `text-2xl font-bold` value, `text-xs` muted label, optional rating pill.
- **Badge**: `rounded-full px-2 py-0.5 text-xs font-medium` with semantic color at 10% opacity background.
- **Button**: Primary (blue-600), Danger (red-600), Secondary (border + bg-secondary). All `rounded-lg px-4 py-2 text-sm font-medium`.
- **Form inputs**: `rounded-lg border border-border bg-bg-secondary px-3 py-2 text-sm` — match button radius.
- **Tabs**: Bottom border indicator (`h-0.5 bg-info`), `text-sm font-medium`, muted when inactive.
- **Progress bars**: `h-1.5 rounded-full` container with color-coded fill. Thin and unobtrusive.
- **Status dots**: Colored circles (`rounded-full`) with aria-label. Green/red/amber/gray.

### Accessibility

- Skip-to-content link (sr-only, visible on focus)
- Focus visible: 2px solid info ring with 2px offset
- ARIA labels on status indicators and interactive elements
- Semantic landmarks: `<nav>`, `<main>`, tab roles with aria-selected/aria-controls
- Color never used as sole indicator — always paired with text labels or icons
- Theme respects `prefers-color-scheme` system preference
