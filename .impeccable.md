## Design Context

### Users
Homelabbers managing personal infrastructure (Proxmox VMs, Docker containers, self-hosted services). Technical enough to run servers, but not professional sysadmins. They want to understand what's happening across their systems without needing to become monitoring experts. The tool should teach and surface insights, not just display data. Commercial expansion is a future goal — the UI must feel professional enough to scale beyond hobbyists.

### Brand Personality
**Reliable, minimal, insightful.**

- **Voice**: Calm, confident, helpful. Speaks like a knowledgeable friend, not a control room.
- **Tone**: Reassuring when things are fine, clear and direct when they're not. Never alarmist.
- **Emotional goal**: The user should feel *in control* — a sense of "I know what's going on" without needing to dig. The UI does the thinking, the user gets the understanding.

### Aesthetic Direction
- **Visual tone**: Clean, polished, minimal. Every element earns its place.
- **Theme**: Dark and light mode, slate-based palette. Semantic colors (emerald/amber/red/blue) for status only — not decoration.
- **Anti-reference**: Grafana. No dense graph walls, no configuration overload, no expert-only interfaces. Insightd is the opposite — curated, opinionated, and immediately legible.
- **Polish direction**: Push current design toward more refined spacing, subtle transitions, and considered micro-details. Professional enough to be commercial, approachable enough for a first-time homelabber.

### Design Principles

1. **Clarity over density**: Show the right information at the right time. One clear insight beats ten raw metrics. Resist the urge to add more graphs — instead, add more meaning.

2. **Calm by default, urgent when needed**: The UI should feel quiet and confident when systems are healthy. Reserve visual intensity (color, motion, prominence) for moments that genuinely need attention.

3. **Teach, don't just display**: Help users understand *why* something matters, not just *what* the number is. Context labels, trend indicators, and plain-language insights over raw data dumps.

4. **Progressive disclosure**: Summaries first, details on demand. Every screen should have a clear hierarchy: hero metric or status, supporting context, then expandable details for those who want to dig deeper.

5. **Polish in the details**: Consistent spacing, smooth transitions, considered typography. The difference between "good enough" and "professional" lives in the micro-details — alignment, hover states, loading states, empty states.
